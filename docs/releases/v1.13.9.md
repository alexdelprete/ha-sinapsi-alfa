# Release v1.13.9

[![GitHub Downloads](https://img.shields.io/github/downloads/alexdelprete/ha-sinapsi-alfa/v1.13.9/total?style=for-the-badge)](https://github.com/alexdelprete/ha-sinapsi-alfa/releases/tag/v1.13.9)

**Release Date:** TBD

**Type:** Patch release - Fixes a v1.13.8 regression that prevented all sensors
from being created when the device reports invalid production data at startup.

## What's Changed

### Fixed

- **Sensor platform no longer crashes when production data is invalid at
  startup** - The v1.13.8 production validity gate withholds the calculated
  energy sensors (`energia_consumata`, `energia_auto_consumata`) from the data
  set until the device reports valid production data. The sensor platform setup
  still accessed those keys directly, raising a `KeyError` that aborted the
  creation of **all** entities — on an Alfa with no production data (or one
  still warming up at Home Assistant startup), the integration came up with no
  sensors at all. The setup now tolerates absent keys and simply skips the
  affected calculated sensors until valid data arrives. Debug log provided by
  [@thomas166](https://github.com/thomas166) in
  [#217](https://github.com/alexdelprete/ha-sinapsi-alfa/issues/217).
